## 功能模块

### 录入数据（Brezi 团队操作）
1. **天猫月结单** — 上传 xlsx，自动解析收款 + 平台费用
2. **Anydoor 对账单** — 上传她每月发来的 xlsx，解析库存消耗
3. **发货台账** — 手动填写每次补货记录（日期/SKU/数量/快递单号）
4. **市场垫付费用** — 手动填写 Brezi 垫付的营销费用

### 看板展示（自动计算）
```
支付宝净收款（天猫收入 - 平台费）
- 机器成本（消耗台数 × ¥1,129.5）
- 配件成本
- 垫付市场费用
= 应打款给 Anydoor
```
与 Anydoor 提交数字对比 → 差异用颜色标注（绿/黄/红）

### 差异判断逻辑
| 差额 | 状态 | 处理 |
|------|------|------|
| < ¥1 | ✅ 一致 | 正常打款 |
| ¥1–¥200 | ⚠️ 小额差异 | 检查月末时间差订单 |
| > ¥200 | ❌ 较大差异 | 需核查库存/收款明细 |

月末时间差：看板会自动标出月底3天内下单、次月才确认收货的订单

## 反馈 → 开发 → 上线流程

```
同事提需求（飞书群 @Elon-bot）
    ↓
Elon 评估可行性 + 排优先级
    ↓
修改代码（/workspace/brezi-dashboard/src/）
    ↓
本地测试（npm run dev）
    ↓
一键部署（vercel --prod）
    ↓
通知群里：已更新，刷新页面查看
```

**提需求格式（建议）：**
> 功能建议：[简述功能]
> 原因：[为什么需要]
> 优先级：高 / 中 / 低

## 注意事项

- 所有财务数据存在 Supabase，不在本地
- xlsx 文件在浏览器解析，不上传任何服务器
- 环境变量（Supabase 密钥）只在 Vercel 后台，不在代码里
- 修改代码后必须 `vercel --prod` 才能上线，本地改了线上不会自动更新
